JavaScript 比较运算符

比较运算符注意事项

  • == (相等): 会进行类型转换
    • 数字和字符串比较时，字符串转为数字
    • 布尔值和其他类型比较时，布尔值转为数字(true=1, false=0)
    • 对象会调用valueOf()或toString()方法转换
    • null == undefined 返回 true
  • === (严格相等): 不会进行类型转换
    • 类型不同直接返回false
    • null === undefined 返回 false
  • != (不等): 会进行类型转换(规则同==)
  • !== (严格不等): 不会进行类型转换(规则同===)
  • >, <,>=, <=: 比较大小
    • 字符串比较按Unicode编码顺序
    • 不同类型比较会尝试转换为数字
    • 任何数与NaN比较都返回false
  • 特殊规则:
    • NaN != NaN (唯一不等于自身的值)
    • +0 === -0

比较运算符示例

const num = 5;
const str = "5";
const bigNum = 10;
const smallNum = 3;

// 1. 相等比较示例
console.log(num == str);   // true - 宽松相等，字符串"5"被转换为数字5
console.log(num === str);  // false - 严格相等，类型不同直接返回false
console.log(num != str);   // false - 宽松不等，等同于!(num == str)
console.log(num !== str);  // true - 严格不等，等同于!(num === str)

// 2. 大小比较示例
console.log(num > smallNum);  // true - 5 > 3
console.log(num < bigNum);    // true - 5 < 10
console.log(num >= 5);        // true - 5 >= 5
console.log(num <= 4);        // false - 5 <= 4

// 3. 类型转换特殊案例
console.log(0 == false);      // true - false被转换为数字0
console.log("" == false);     // true - 空字符串和false都被转换为0
console.log(null == undefined); // true - 特殊规则，但null/undefined不与其他值相等
console.log(" \t\r\n" == 0);  // true - 空白字符串被转换为0
console.log(NaN == NaN);      // false - NaN是唯一不等于自身的值

// 4. 字符串比较规则
console.log("a" < "b");       // true - 按字符Unicode编码比较，a(97) < b(98)
console.log("2" > "12");      // true - 字符串比较是按位比较，"2"(50) > "1"(49)
console.log("5" >= 5);        // true - 字符串"5"被转换为数字5
